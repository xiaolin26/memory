# Memory 系统 - 通用模板

> 一套完整的项目记忆和开发规范体系，帮助团队高效协作

---

## 🎯 核心价值

- 📝 **记录一切**：技术决策、变更原因、废弃方案
- 🤖 **AI 友好**：让 AI 快速理解项目
- 👥 **团队协作**：统一开发规范
- 🔄 **持续更新**：项目发展，文档同步

## 🚀 如何使用

### 第1步：复制模板

```bash
# 复制到项目
cp -r [模板目录]/* [项目目录]/
```

### 第2步：替换占位符

- `[项目名称]` → 实际项目名
- `[项目路径]` → 实际路径
- 其他 `[占位符]` → 实际内容

### 第3步：根据项目调整

- 填写核心原则
- 填写技术栈
- 填写项目结构

---

## ✅ 最小实施

**只需要做 3 件事**：

1. **复制文件**（5分钟）
   ```bash
   cp CONSTITUTION.md [项目目录]/
   cp MEMORY.md [项目目录]/
   ```

2. **填写核心信息**（10分钟）
   - CONSTITUTION.md：核心原则 + 技术栈
   - MEMORY.md：项目结构 + 技术栈

3. **开始使用**（持续）
   - 开发前：读 MEMORY.md + CONSTITUTION.md
   - 开发后：更新 MEMORY.md

## 📝 核心文件说明

### CONSTITUTION.md（项目宪法）

**作用**：定义开发规范和约定

**核心内容**：
- 5 条核心原则
- 敏感信息管理
- 版本号管理
- 提交规范
- 禁止事项

**何时更新**：仅在调整规范时（需团队讨论）

---

### MEMORY.md（项目记忆）

**作用**：记录项目历史和技术决策

**核心内容**：
- 按时间顺序的变更记录
- 技术栈总结
- 核心设计原则
- 重要技术决策（当前 + 废弃）
- 待解决问题

**何时更新**：每次重大变更后

## 🆘 常见问题

**Q: 必须使用所有文件吗？**

A: 不必须。核心是 CONSTITUTION.md 和 MEMORY.md。

**Q: 会增加工作量吗？**

A: 初期 15-20 分钟填写，之后每次变更 3-5 分钟更新。长期节省大量时间。

**Q: 适用于所有项目吗？**

A: 适用于大多数软件项目（Web、移动端、小程序等）。

**Q: 如何确保团队遵循？**

A: 培训 + Code Review + 定期检查。
